WCAG 2.2 AA — Testable Criteria
Accessibility Requirements
must
Color contrast ≥ 4.5:1 for body text, ≥ 3:1 for large text
Test: lime (#abf600) on black (#000) = 8.59:1 ✓ · white on black = 21:1 ✓ · lime on white = 1.7:1 — do not use for body text
must
Focus-visible ring must be 3px solid color.surface.raised (#abf600) on all interactive elements
Test: Tab through page — every element must show visible focus indicator. Use :focus-visible not :focus.
must
Buttons must have accessible name via aria-label or visible text
Test: Axe DevTools — zero "button name" violations. Icon-only buttons require aria-label.
must
Error messages must use role="alert" and reference input via aria-describedby
Test: Trigger error state — screen reader must announce error without user action.
must
Touch targets must be minimum 44×44px
Test: Mobile emulation → Lighthouse audit → no small tap target warnings.
must
Status indicators must not rely on color alone
Test: Simulate color blindness filter — Active/Idle/Error must remain distinguishable via shape, label, or pattern.
should
Loading states should include aria-busy="true" and sr-only status text
Test: Trigger loading → screen reader should announce "Loading…" without visual dependency.
should
Cards should use article or section with heading hierarchy
Test: Headings map (Axe) — card titles appear at correct h-level.
never
Never use lime (#abf600) as text on white background — contrast fails at 1.7:1
Lime is for backgrounds and accents only. Text on lime must be black (#000).
never
Never hide focus outline or set outline: none without focus-visible replacement
Test: Keyboard-only navigation — every focused element is visually distinct.
Anti-patterns
Prohibited Implementations
✕
Raw hex values in component code — use semantic tokens only
Bad: background: #abf600 · Good: background: var(--color-surface-raised)
✕
One-off spacing values — no ad-hoc margins like margin: 13px
Use space.* tokens only: space.1 through space.8
✕
Ambiguous button labels — "Click here", "Submit", "OK"
Use action + object: "Earn Rewards", "Connect Wallet", "View History"
✕
Font sizes below 9px (font.size.xs) in any visible element
Minimum: font.size.xs = 9px for labels/tokens, font.size.md = 14px for interactive
✕
Gradient or blur backgrounds — flat neubrutalism only
No box-shadow with blur radius. Shadows are solid offset only (shadow.1, shadow.2).
✕
Shipping any interactive component without all 7 states defined
Required: default, hover, focus-visible, active, disabled, loading, error

Content Standards
✓ Concise, confident, present tense
✓ Action + object labels: "Earn Rewards" not "Rewards"
✓ Numbers formatted with commas: 12,840 not 12840
✓ Timestamps in UTC with relative label: "Today, 14:32 UTC"
✓ Error messages: specific cause + resolution path
✗ No passive voice in CTAs
✗ No placeholder text as label substitute
✗ No "Click here" or "Learn more" without context
